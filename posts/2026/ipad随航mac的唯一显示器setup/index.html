iPad 随航作为 Mac 唯一显示器的 Setup

2026/01/09 MacOS/ IPad/ Sidecar/ Automation

Why#

  • 我有 Mac mini M4 和 iPad Air 5，没有 MacBook
  • 有多位置使用的需求
  • 在 B 站和 YouTube 上看过相关做法，但在 macOS 26 上不能直接复现，想自己折腾

Preconditions#

  • Mac 与 iPad 登录同一 Apple ID，且开启双重认证
  • Wi-Fi 与蓝牙保持开启，Handoff (接力) 已启用
  • 如果用线连接: USB-C/雷雳线质量要稳定

How#

1. 用 Automator 做一个 开启随航 的小应用#

a. 打开 Automator，新建 Application，搜索并添加 Run AppleScript
b. 粘贴下面代码，把 目标iPad 改成你的 iPad 名称:

-- 找到目标 iPad 的最后一个匹配索引
set targetIndex to my findLastTargetIndex("目标iPad", menuItems)
展开查看完整代码
-- 定义查找列表索引的函数
on findLastTargetIndex(targetItem, itemList)
	set lastIndex to 0
	repeat with i from (count of itemList) to 1 by -1
		if item i of itemList is targetItem then
			set lastIndex to i
			exit repeat
		end if
	end repeat
	return lastIndex
end findLastTargetIndex

-- 播放提示音
do shell script "afplay /System/Library/Sounds/Pop.aiff"

tell application "System Settings"
	activate
	do shell script "afplay /System/Library/Sounds/Pop.aiff"
	delay 1
	
	tell application "System Events"
		tell process "System Settings"
			
			-- 获取系统语言
			set systemLanguage to do shell script "defaults read -g AppleLocale"
			
			if systemLanguage starts with "zh" then
				if systemLanguage is "zh_CN" then
					set menuItemName to "显示器"
					set menuBarItemName to "显示"
					set addButtonName to "添加"
				else
					set menuItemName to "顯示器"
					set menuBarItemName to "显示方式"
					set addButtonName to "加號"
				end if
			else if systemLanguage starts with "en" then
				set menuItemName to "Displays"
				set menuBarItemName to "View"
				set addButtonName to "Add"
			end if
			
			-- 打开显示器菜单
			click menu item menuItemName of menu menuBarItemName of menu bar item menuBarItemName of menu bar 1
			delay 0.3
			
			-- 最新 macOS 结构调整
			tell group 1 of group 3 of splitter group 1 of group 1 of window menuItemName
				try
					delay 1 -- 等待界面加载
					
					-- 点击第一个菜单按钮 (替代原 pop up button)
					click menu button 1
					
					-- 获取菜单项
					set menuItems to name of menu items of menu 1 of menu button 1
					
					-- 找到目标 iPad 的最后一个匹配索引
					set targetIndex to my findLastTargetIndex("目标iPad", menuItems)
					
					-- 点击目标菜单项
					delay 0.3
					click menu item targetIndex of menu 1 of menu button 1
					
					do shell script "afplay /System/Library/Sounds/Blow.aiff"
				on error
					delay 0.5
					do shell script "afplay /System/Library/Sounds/Hero.aiff"
				end try
			end tell
		end tell
	end tell
end tell

delay 1
tell application "System Settings" to quit

2. 配快捷键触发#

a. 保存到 文档，命名为 开启随航
b. Automator 新建 Quick Action，搜索并添加 Launch Application
c. Workflow receives current 选择 no input，在 Launch Application 里点 Other...，选择刚保存的 开启随航，保存为 启动随航
d. 系统设置 -> 键盘 -> 快捷键 -> 服务 -> 通用 -> 启动随航，设置固定快捷键 (例如 Control+Command+C)
e. 第一次触发会提示权限，在 隐私与安全性 -> 辅助功能 中勾选 开启随航

3. 让 macOS 认为有显示器#

a. 下载 BetterDisplay，创建一个 Virtual Display，比例选择 iPad 型号，并设为开机启动
b. 确认 Mac mini 的 Wi-Fi/蓝牙保持开启 (有线直连也需要)

4. 无屏开机的操作流程#

a. 听到开机声后等待 5 秒，盲打密码回车，等 20-30 秒进入桌面
b. 按快捷键触发 启动随航，听到提示音说明运行成功
c. 第一次成功连接后，在 系统设置 -> 显示器 中把 iPad 设为镜像虚拟屏幕，并把虚拟屏幕设为主显示器

Method#

  • macOS 必须检测到已点亮的显示器才能启用随航 -> 用 BetterDisplay 模拟虚拟屏
  • 黑屏无法手动点开随航 -> 用 Automator + AppleScript 直接操控系统设置
  • 黑屏也要触发动作 -> 用 Quick Action 绑定固定快捷键

Final#

  • 现在开机后盲打密码 -> 快捷键 -> iPad 上线，基本就能把 iPad 当作唯一显示器用，ipad本身也可以单独使用
  • ipad加上mac mini和键盘鼠标出门的话一个普通的电脑包也装的下，基本完美
