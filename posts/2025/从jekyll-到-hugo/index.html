从 Jekyll 到 Hugo

2025/12/23 X

初衷与起点#

我最初选择 Jekyll，是因为它是经典的静态站点生成器，和 GitHub Pages 的组合非常常见。
博客的目标也很简单：记录发生在我身上的事，分享想法与价值。

Jekyll 让我学到的事#

Jekyll 的流程清晰，但依赖 Ruby 生态，本地环境并不“轻”：

  • Ruby / RubyGems / Bundler 版本要对齐
  • 常用命令是 bundle install、bundle exec jekyll serve
  • GitHub Pages 部署还要处理 SSH Key、Git 配置、.DS_Store 等问题

我还遇到过网络与环境问题，例如 DNS 污染、Homebrew 源切换、Ruby 版本切换（RVM/3.2.2）等。
这些坑我都一步步解决了，也因此更理解了 Jekyll 的生态。

为什么切换到 Hugo#

我决定迁移到 Hugo，核心原因很明确：

  1. Jekyll 比较老，对我来说维护成本偏高。
  2. Hugo 官方支持 Docker，本地预览不再依赖复杂的环境配置。
  3. 支持双仓布置：私有仓放完整源码，公开仓只放静态网页，结构更清爽也更安全。

我使用的工具和方法#

我主要是使用的是chagpt，没会员的时候用 gemini 和 grok，从 chatgpt3.5（没开会员）到 chatgpt4（开了 plus 会员）到 4o，直到现在最新版本 vscode 里面的 codex（GPT-5.2-Codex Extra high 模式），越来越好用的 ai 真的让我这个小白实现了几乎免费的建站梦。（其他段落均用 codex 总结的）

迁移后的结构与体验#

迁移到 Hugo 后，构建速度明显更快，配置也更直观。

Final#

Jekyll 帮我迈出了第一步，而 Hugo 让我更专注于写作本身。
从环境折腾转向内容表达，是这次迁移最大的价值。
